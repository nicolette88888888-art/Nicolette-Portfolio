## üìù Notes

- **Performance First**: Always test animations for performance
- **Accessibility**: Respect user preferences for reduced motion
- **Mobile**: Ensure animations work well on touch devices
- **Progressive Enhancement**: Core functionality should work without animations
- **Testing**: Test on various devices and browsers
- **Balance**: Don't overdo it - subtlety is key

---

## üé® Design Philosophy

1. **Subtlety Over Flashiness** - Animations should enhance, not distract
2. **Performance Matters** - Smooth 60fps animations
3. **User Control** - Respect user preferences
4. **Purposeful** - Every animation should have a reason
5. **Consistent** - Maintain design language throughout
6. **Accessible** - Don't exclude users with motion sensitivity

---
